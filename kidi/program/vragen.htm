Berekeningen zijn niet genoeg

Met simpele wiskundige formules kom je heel ver, maar er zijn vaak punten waar je vast komt te zitten als je alleen maar berekeningen op variabelen blijft doen. Denk bijvoorbeeld aan delen door 0 bij de wiskunde, bij het programmeren zou het fijn zijn als je zulke gevallen kon voorkomen. Gelukkig is dat natuurlijk mogelijk, in de vorm van if/ifelse/else constructies.

De theorie is dat je verschillende vragen stelt aan de variabelen. Is de gekozen kleur blauw? Is de leeftijd boven de 12? Is de gast akkoord gegegaan met de algemene voorwaarden? Als dat zo is voer je een aantal berekeningen en acties uit, zo niet een andere reeks berekeningen en acties. Het is zelfs mogelijk om een tussenvraag te stellen. Bijvoorbeeld is de gast boven de 16, dan mag hij zijn eigen email gebruiken, als hij boven de 12 of 12 is moet hij een email van zijn ouders opgeven en onder de 12 mag je helemaal de website niet op.

De simpelste vraag: If met else

Een vraag heeft altijd een bool als antwoord, en die bool moet je dan invullen in de if constructie. Als de bool true is wordt de code die je zelf kan bepalen dan uitgevoerd, als hij false is wordt hij niet uitgevoerd, en wordt eventueel de code in het else-deel uitgevoerd als je die hebt aangemaakt. 

string Tekst = "Toegang: ";
int Leeftijd = 13;
bool BovenDeTwaalf = (Leeftijd > 11);
if (BovenDeTwaalf)
{
   Tekst += "Gegeven";
}
else
{
   Tekst += "Geweigerd";
}

Wat { hiertussen } staat wordt dus uitgevoerd wanneer de if of de else klopt. Dit ziet er ingewikkeld uit en dat komt misschien omdat ik het er ingewikkeld uit heb laten zien, want dit zou precies hetzelfde doen: 

string Tekst = "Toegang: Geweigerd";
int Leeftijd = 13;
if (Leeftijd > 11)
{
   Tekst = "Toegang: Gegeven";
}

Wat je nu denkt klopt, je kan inderdaad de vraag meteen in de if invullen, je hoeft het resultaat niet eerst op te slaan als een bool variabel.

Je hebt verschillende vergelijkende tekens, de meest gebruikte zijn:

  • '<': Links is kleiner dan rechts
  • '>': Rechts is kleiner dan links
  • '==': Links en rechts zijn hetzelfde, werkt met alle typen variabelen
  • '!=': Links en rechts zijn niet hetzelfde, werkt met alle typen variabelen
  • '<=': Links is kleiner dan rechts of ze zijn hetzelfde
  • '>=': Rechts is kleiner dan links of ze zijn hetzelfde

Tip: (Var < 5) en (Var <= 4) zijn dus precies hetzelfde als het om hele getallen gaat.

Vragen in vragen

Soms heb je meer criteria waarvan je wilt dat ze allemaal goed zijn voordat je een stuk code wilt uitvoeren. Bijvoorbeeld het registreren van een email adres, dan wil je zeker weten dat die email niet al gebruikt wordt, de naam niet te lang of te kort is, er geen persoonlijke informatie inzit enzovoort. Daarvoor heb je of heel veel vragen nodig in elkaar, of je gebruikt de 'of' en 'en' tekens: || en &&. Meerdere vragen sorteren kan je net als in de wiskunde met de simpele ronde haakjes ( ) om verwarring te voorkomen. Bijvoorbeeld op een forum voor alleen tieners: 

string Tekst = "Welkom!";
int Leeftijd = 16;
bool Administrator = false;
if (Leeftijd < 13 || Leeftijd > 18)
{
   if (Administrator == false)
   {
      Tekst = "Je mag hier niet komen, dit is alleen voor tieners";
   }
}

Dit kan je inkorten tot: 

string Tekst = "Welkom!";
int Leeftijd = 16;
bool Administrator = false;
if ((Leeftijd < 13 || Leeftijd > 18) && Administrator == false)
{
   Tekst = "Je mag hier niet komen, dit is alleen voor tieners";
}

If / elseif (/ elseif / elseif ...) / else

In één vraagblok kan je heel veel subvragen stellen waarvan de code alleen uitgevoerd wordt als de vragen ervoor allemaal niet klopten: 

string LeeftijdsFase = "Onbekend";
int Leeftijd = 25;
if (Leeftijd > 60)
{
   LeeftijdsFase = "Oud";
}
else if (Leeftijd >= 18)
{
    LeeftijdsFase = "Volwassen";
}
else if (Leeftijd > 12)
{
    LeeftijdsFase = "Tiener";
}
else
{
   LeeftijdsFase = "Kind";
}

Dit is natuurlijk ook samen te voegen met wat je geleerd hebt in de column hiernaast.
