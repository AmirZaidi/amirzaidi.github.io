De logica achter programmeertaal

Programmeren is meer dan een hobby of baan. Het is een gecompliceerde manier van denken, de meest logische manier van denken die er bestaat. Hij is zo logisch, dat als je één fout maakt je hele brein, of de CPU van een computer, vol kan komen met fouten. Als je het kan zonder fouten kan je het zo efficient maken als je wilt. Een brein heeft de mogelijkheid om vele gedachtes en acties tegelijk te behandelen, een computer kan niet veel berekeningen tegelijk doen, maar kan er wel veel meer per seconde, en de bedoeling van een programmeur is die rekenkracht zo goed mogelijk te gebruiken in zijn programma. Als je wilt leren hoe je dat moet doen, ben je hier op de juiste plaats.

Een stap verder dan het statische

Zover hebben we alleen nog maar HTML en CSS behandeld. HTML en CSS zijn statische talen, dat wil zeggen dat je er geen logica of vraag in kan zetten. Je zegt wat waar moet komen op de pagina, en je browser weet hoe het eruit moet zien. 

<div style="background: red; position: absolute; top: 50px; left: 50px;">
   Hallo!
</div>

Je kan interactiviteit toevoegen met bijvoorbeeld Javascript, wat een webpagina levendiger kan maken met wat berekeningen en effecten. Javascript is gebaseerd op Java, wat één van de bekendste programmeertalen is om programma's in te maken. Java, en andere talen zoals C en de varianten daarop (C++, C#), Visual Basic en PHP zijn talen waarin je berekeningen kan maken en je een dynamische reactie kunnen geven, gebaseerd op wat je invoert. Bijvoorbeeld een persoonlijk ID nummer maken van een naam.

  • Het programma vraagt om een naam
  • De naam wordt omgekeerd met een simpele opdracht ter beveiliging
  • Alle letters in de naam worden met een ascii-tabel omgezet in cijfers
  • De cijfers worden in een cryptografische manier aan elkaar geplakt, bijvoorbeeld door bij 105 en 203 de 5 en 2 op te tellen zodat je 10703 krijgt, dat is weer ter beveiliging van de originele naam

Op die manier kan je een programma gebruiken om een moeilijke taak voor je te doen, zodat je moeilijke berekeningen niet zelf hoeft te doen. Met programmeertalen kan je nog veel meer dan alleen berekeningen, maar die functies liggen eraan over welke taal je het hebt. C# is een taal om .exe (executable programs) te maken voor Windows, PHP is een taal bedoeld om dynamische HTML pagina's mee te maken. Dus C# heeft functies zoals het openen en sluiten van een ander programma, en PHP kan bijvoorbeeld de invoer van je naam en wachtwoord lezen wanneer je inlogt op een PHP pagina.

Voorbeelden in C#

Op de volgende pagina gaan we echt beginnen aan programmeren. Vanaf daar zal ik al mijn voorbeelden in C# geven omdat je daar heel veel in kan, de syntax makkelijk te begrijpen is en ik daar het meeste ervaring mee heb.

Gebruik een IDE

Om C#, PHP of een andere taal te gebruiken is het zeer handig, vaak ook nodig, om een goede IDE ervoor te downloaden en installeren. Voor C# is de beste en meestgebruikte IDE Visual Studio, een programma gemaakt door Microsoft zelf om mensen makkelijk C# programma's te laten maken. Visual Studio Express is de gratis versie ervan en kan je downloaden vanaf de officiële microsoft site.

Als je de voorbeelden zelf wilt uitproberen en er mee wilt experimenten raad ik je zeer aan om dit programma te installeren en te gebruiken. Wanneer je het geinstalleerd hebt moet je het openen, en dan op nieuw project klikken. Kies een C# Console Application. Er komt al heel wat code op je beeld, die je nog een lange tijd niet hoeft te gebruiken of veranderen. Waar je jouw test code gaat zetten is hiertussen: 

static void Main(string[] args)
{
   -- Zet hier je code --
}

Daarna hoef je alleen op F5 te drukken en je code zal worden uitgevoerd. Maar aangezien je nog niet hebt geleerd om echt iets nuttigs te doen is het niet echt nodig om al zo ver te gaan.

Verschillende Syntaxen

In de meeste talen kan je niet de code kopieren tussen verschillende talen, omdat sommige dingen anders heten of je ze anders schrijft. Bijvoorbeeld het dollarteken is in PHP heel veel nodig, terwijl je ermee in C# alleen maar errors gaat veroorzaken. Om aan een nieuwe taal te leren moet je ook goed worden met de syntax, en aan veel talen tegelijk beginnen gaat je alleen maar verwarren omdat de syntaxen hoogstwaarschijnlijk op veel plaatsen verschillen.
