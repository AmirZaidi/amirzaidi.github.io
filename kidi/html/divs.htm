Box theorie

Als je gaat kijken naar de code van de meeste HTML websites, kun je zien dat deze vooral bestaat uit <divs>. Met deze divs wordt de layout van de website aangegeven.
Deze pagina is ook opgebouwt met meerdere divs. De wat blauwere balk bovenin is een div voor de navigatie, het midden van de pagina is ook een div, de content kan niet buiten de rand van deze div komen. Daarom zijn er de lege balken aan allebei de kanten.

HTML layout tags

In html5 zijn er een aantal nieuwe tags toegevoegd die je kan gebruiken voor een layout, in plaats van divs met classes en id's te gebruiken.
Deze nieuwe tags zijn <header>, <nav>, <section> <article>, <aside>, <footer>, <details> en <summary>. Deze doen allemaal precies zoals hun naam aangeeft. De manier waarop deze stukken eruit zien kan je aanpassen met CSS, ook kun je hierin aanpassen hoe de content hierin eruit moet zien in CSS. Hoe je dit doet leg ik uit in het volgende hoofdstuk.

Align

Divs hebben zelf één eigen attribuut. Dit is het attribuut align: . Hiermee geef je aan waar de tekst in de div moet uitlijnen. Als values hiervoor kan je top, right, bottom en left invoeren. Deze verplaatsen de tekst respectievelijk naar boven, rechts, onder en links. Het is een beetje te vergelijken met het uitlijnen in microsoft word. Bij align="right" komt het einde helemaal rechts, hoe meer tekst er is, hoe verder het begin van de tekst naar links gaat.

Werken met divs

Als je toch liever met divs wilt werken, of je gebruikt een oudere versie van html dan html5, is er de <div>.
Als je alleen een div zou invoeren in een html bestand, alles dat je dan nest in deze div-tag wordt gewoon op de website geplaatst alsof het gewoon los zou staan. Als je wilt dat de box zelf eranders uit komt te zien gaat dit met het zogenaamde box model. Dit staat uitgelegd bij de paragraaf boxen bij CSS. Om de stijl van de content in een div te veranderen gebruik je combinators, deze staan uitgelegd bij speciale eigenschappen.
